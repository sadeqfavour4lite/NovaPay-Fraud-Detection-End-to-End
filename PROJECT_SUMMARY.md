# Project Summary

NovaPay Fraud Detection is an end-to-end machine learning project for identifying fraudulent digital money-transfer transactions. It covers data cleaning, feature engineering, exploratory analysis, baseline models, advanced imbalance-aware modelling, explainability, and FastAPI deployment.

## Business Objective

Detect suspicious transactions early enough for manual review or hold decisions while preserving a smooth experience for legitimate customers.

## Workflow

1. `03_cleaning.ipynb` cleans raw transactions and saves `data/processed/cleaned_transactions.csv`.
2. `04_features_eda.ipynb` builds behavioural, device, corridor, velocity, and time features.
3. `05_baseline_models.ipynb` trains baseline classifiers and saves baseline metrics/artifacts.
4. `06_advanced_models.ipynb` trains advanced models with imbalance handling and saves the best model.
5. `07_explainability.ipynb` creates transaction-level explanations and stakeholder review templates.
6. `08_api_pipeline.ipynb` documents API packaging, validation, local scoring, and Docker deployment.

## Key Engineering Choices

- Chronological train/test split to mimic production scoring on future transactions.
- Preprocessing inside model pipelines to reduce leakage risk.
- Fraud-focused metrics such as recall, precision, F1, ROC AUC, and average precision.
- Threshold-based decisioning for `approve`, `manual_review`, and `hold_and_investigate`.
- FastAPI service with Pydantic validation and explicit model-load health checks.

## Repository Layout

- `notebooks/`: original project notebooks, renamed consistently.
- `data/raw/`: source NovaPay transaction dataset.
- `data/processed/`: cleaned dataset and generated feature datasets.
- `src/novapay/`: reusable cleaning, feature engineering, and path utilities.
- `api/`: production-style fraud scoring API.
- `models/`: saved model artifacts.
- `reports/`: figures, metrics, templates, and API validation artifacts.
- `tests/`: regression tests for reusable project code.
